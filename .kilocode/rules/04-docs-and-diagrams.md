# Documentation & Diagrams (Mermaid)

- Prefer **Mermaid** diagrams embedded directly in Markdown (README, docs/*).
- Store examples/drafts in `docs/diagrams/*.mmd`; keep at least one key diagram inline in README.
- Supported types: flowchart, sequence, state, class, er, gantt, etc.
- Keep diagrams simple; use meaningful labels and `subgraph` for domains.
